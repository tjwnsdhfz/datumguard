# DatumGuard v0.4.0 Launch Kit

이 문서는 `v0.4.0`의 검증된 사실만 사용한 소프트 런치 자산이다. Show HN/Product Hunt 같은 대규모
cross-post는 [`growth-and-launch-roadmap.md`](../growth-and-launch-roadmap.md)의 전체 launch gate가
닫힌 뒤 진행한다.

## One-line positioning

> DatumGuard is a fail-closed CAD assurance harness that reopens and remeasures generated or uploaded artifacts before it exposes an approved export.

한국어:

> DatumGuard는 AI/CAD가 만든 결과를 그대로 믿지 않고, 저장된 artifact를 다시 열어 재측정한 뒤 검증된 export만 허용하는 fail-closed 공학설계 보증 하네스입니다.

## 한국어 소프트 런치 글

AI로 CAD 객체를 빠르게 만드는 것보다 어려웠던 문제는 “요구한 좌표와 치수가 실제 저장 파일에도
정확한가?”를 증명하는 일이었습니다.

DatumGuard v0.4.0은 요구조건을 잠근 contract로 만들고, DXF를 생성한 뒤 별도 reader로 다시 열어
0.001mm 기준으로 재측정합니다. 외부 DXF는 entity family를 `MEASURED`, `RENDER_ONLY`,
`UNSUPPORTED`로 분류하며, 검증 범위가 불완전하면 geometry equivalence를 주장하지 않습니다.

이번 release에는 Rhino 8 + Grasshopper + Cordyceps에서 실제 생성한 6개 Rhino object GUID,
교환/contract/artifact hash, DXF, verification JSON을 묶은 round-trip evidence도 포함했습니다.

- Live demo: https://datumguard-tjwnsdhfz.vercel.app/?utm_source=github&utm_medium=discussion&utm_campaign=v0_4_launch
- 3-minute case study: https://datumguard-tjwnsdhfz.vercel.app/case-study?utm_source=github&utm_medium=discussion&utm_campaign=v0_4_launch
- Reproducible release: https://github.com/tjwnsdhfz/datumguard/releases/tag/v0.4.0

현재 범위는 설계 artifact 정확성 보증과 제한된 2D frame screening입니다. 구조 안전 인증, 법규
적합성 또는 전문 엔지니어 승인을 대체하지 않습니다. Rhino/GH 또는 실제 외부 DXF에서 재현되지 않는
사례가 있다면 Discussion에 파일 형식과 실패 조건을 알려주세요.

## English soft-launch post

I built DatumGuard after finding that generating CAD geometry with an agent was easier than proving that the
serialized artifact still matched the intended coordinates and dimensions.

DatumGuard v0.4.0 locks requirements into a contract, writes DXF, reopens it through an independent reader,
and remeasures it before allowing an approved export. Uploaded DXF entities are classified as `MEASURED`,
`RENDER_ONLY`, or `UNSUPPORTED`; incomplete coverage fails closed instead of reporting geometry equivalence.

The release also includes a real Rhino 8 + Grasshopper + Cordyceps round trip with 6 Rhino object GUIDs,
exchange/contract/artifact hashes, DXF, verification JSON, two CycloneDX SBOMs, and exact production deployment
evidence.

- Live: https://datumguard-tjwnsdhfz.vercel.app/?utm_source=github&utm_medium=discussion&utm_campaign=v0_4_launch
- Case study: https://datumguard-tjwnsdhfz.vercel.app/case-study?utm_source=github&utm_medium=discussion&utm_campaign=v0_4_launch
- Release: https://github.com/tjwnsdhfz/datumguard/releases/tag/v0.4.0

This is engineering design assurance and preliminary 2D frame screening—not structural certification,
code compliance, or professional approval. I would value reproducible Rhino/Grasshopper and external-DXF
edge cases.

## 포트폴리오 bullet

- 자연어 추정 대신 versioned `DesignContract`와 datum/locked dimension을 도입해 공학설계 입력을 구조화했다.
- DXF writer와 reader/verifier를 분리하고 serialized artifact를 0.001mm 기준으로 재측정해 미검증 export를 차단했다.
- 외부 DXF entity coverage를 `MEASURED`/`RENDER_ONLY`/`UNSUPPORTED`로 분류하고 nested block, cycle,
  complexity budget까지 fail-closed하는 completeness gate를 설계했다.
- Rhino 8·Grasshopper·Cordyceps 실제 왕복에서 6개 object GUID와 XDATA, contract/artifact hash를 연결했다.
- Architecture, piping, plate, 2D frame, external artifact audit를 하나의 stateless FastAPI/MCP core와
  15-page Next.js 제품으로 통합했다.
- 413 pytest, 41 real-API Playwright, CodeQL, dependency audit, container/SBOM/Trivy와 exact-SHA production
  smoke를 배포 gate로 구성했다.
