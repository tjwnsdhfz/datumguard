# DatumGuard 사용자 시나리오와 사용성 기준

이 문서는 Architecture workspace의 기능 목록이 아니라 사용자가 완료해야 하는 작업을 기준으로 UX를 정의한다. 제품의 핵심 약속은 `정확한 값 입력 → 독립 재측정 → 검증된 산출물만 export`이며, 구조·안전·법규 적합성 판정은 제공하지 않는다.

## 현재 사용성 진단

- 첫 화면이 전문 CAD 작업대로 바로 열려 처음 방문한 사용자가 시작 순서를 알기 어렵다.
- `Wall`, `Column`, `Door`, `Window` 도구는 실제 객체 생성 동작 없이 선택 상태만 바꾸므로 잘못된 기대를 만든다.
- 수치 입력이 키 입력마다 contract에 반영되어 중간값과 빈 문자열이 undo 기록과 검증 상태를 오염시킬 수 있다.
- 검증 결과가 화면 아래에 나타나지만 완료 후 결과로 이동하거나 다음 행동을 안내하지 않는다.
- 실패 violation은 객체 ID만 보여주며 해당 객체 선택, 수정 위치, 재검증으로 연결되지 않는다.
- 900px 미만에서는 캔버스 편집과 객체 트리가 모두 비활성화되어 다른 객체를 선택할 수 없다.
- 복원된 IndexedDB draft가 조용히 적용되어 현재 화면이 예제인지 이전 작업인지 구분하기 어렵다.

## 핵심 사용자 시나리오

### `DG-UX-001` 처음 방문한 사용자가 정상 샘플을 검증한다

**사용자**: CAD 자동 검증을 처음 접한 메이커·공학 학습자

**목표**: 제품의 핵심 동작을 1분 안에 직접 확인한다.

1. 첫 화면에서 `샘플 선택 → 정확한 값 확인 → DXF 검증` 순서를 확인한다.
2. `정상 샘플`을 선택한다.
3. 현재 선택 객체와 주요 mm 값을 확인한다.
4. 한 개의 주 행동 버튼으로 DXF 생성·재측정을 실행한다.
5. 완료 후 결과 영역으로 이동해 `VERIFIED / PASS`와 다운로드 가능 상태를 확인한다.

**완료 기준**

- 다음 행동이 항상 하나의 주 버튼으로 보인다.
- 검증 완료 시 결과 제목에 키보드 초점이 이동하고 screen reader live region이 상태를 알린다.
- DXF·PDF·verification JSON을 포함한 ZIP은 PASS 상태에서만 활성화된다.

### `DG-UX-002` 정확한 치수를 안전하게 수정한다

**사용자**: 좌표와 치수를 직접 확인하려는 공학 학습자

**목표**: 한 객체의 값을 의도한 숫자로 변경하고 필요하면 되돌린다.

1. 모델 트리 또는 모바일 객체 선택기에서 객체를 선택한다.
2. inspector에서 수치를 편집한다.
3. `Enter` 또는 focus 이동으로 값을 확정한다.
4. 잘못된 값은 inline 오류로 확인하고 contract에는 반영하지 않는다.
5. `Escape`로 편집 전 값으로 돌아가거나 `Undo`로 확정 변경을 되돌린다.

**완료 기준**

- 입력 중간 문자열은 contract와 undo history에 기록되지 않는다.
- 한 번의 확정은 undo history 한 단계만 추가한다.
- 길이·두께·폭·깊이는 0보다 커야 하며 offset은 0 이상이어야 한다.
- input 안에서 `Ctrl/Cmd+Z`와 `Space`를 사용해도 캔버스 history 또는 pan 상태를 변경하지 않는다.

### `DG-UX-003` 실패 원인을 찾고 300mm 외벽 간격을 복구한다

**사용자**: 검증 실패와 export 차단을 확인하려는 포트폴리오 검토자·학습자

**목표**: 실패가 어떻게 추적되고 수정되는지 확인한다.

1. `300 mm open loop` 실패 샘플을 선택한다.
2. 검증을 실행해 `DG_ARCH_EXTERIOR_OPEN`과 export 차단을 확인한다.
3. violation의 `객체 선택`을 눌러 `wall-north` inspector로 이동한다.
4. `End X`를 `0 mm`로 수정하거나 안내된 `300 mm 간격 닫고 재검증`을 실행한다.
5. 재검증 PASS와 활성화된 다운로드를 확인한다.

**완료 기준**

- violation은 관련 객체 ID를 선택 가능한 버튼으로 제공한다.
- 데모 전용 복구는 수정할 값과 객체를 버튼 문구에 명시한다.
- 실패 중 다운로드는 계속 비활성화된다.
- 복구 후 동일 contract 경로로 다시 검증한다.

### `DG-UX-004` 이전 draft를 복원하거나 정상 샘플로 돌아간다

**사용자**: 같은 브라우저로 다시 방문한 사용자

**목표**: 복원된 상태를 인지하고 안전하게 계속하거나 초기화한다.

1. IndexedDB에 저장된 architecture draft가 있으면 복원 사실을 확인한다.
2. 기존 값을 계속 편집하거나 `정상 샘플로 복원`한다.
3. 샘플 복원도 undo 가능한 한 단계로 기록한다.

**완료 기준**

- 복원 여부와 로컬 저장 범위를 화면에 표시한다.
- 서버 계정이나 서버 프로젝트 저장으로 오해할 표현을 사용하지 않는다.
- 정상 샘플 복원은 다른 저장소 데이터나 서버 상태를 변경하지 않는다.

### `DG-UX-005` 태블릿·모바일에서 수치 입력과 검증을 완료한다

**사용자**: 900px 미만 화면에서 결과를 확인하는 사용자

**목표**: drag 없이 객체를 선택하고 정확한 값을 입력해 검증·다운로드한다.

1. 정밀 drag가 데스크톱 전용임을 확인한다.
2. 모바일 객체 선택기에서 column, wall, opening, grid를 선택한다.
3. inspector에서 수치를 확정한다.
4. 화면 하단의 상태 기반 고정 버튼으로 검증하거나 다운로드한다.

**완료 기준**

- 375px와 820px에서 가로 overflow가 없다.
- 객체 트리가 숨겨져도 모든 편집 가능 객체를 선택할 수 있다.
- 주 행동 버튼은 `준비 중`, `검증`, `복구`, `다운로드` 상태를 명확히 구분한다.

### `DG-UX-006` 구현 범위를 오해하지 않고 다른 설계 영역을 탐색한다

**사용자**: 채용 담당자·포트폴리오 검토자

**목표**: 구현된 기능과 후속 범위를 빠르게 구분한다.

1. 현재 workspace가 Architecture임을 확인한다.
2. 실제 동작하는 `Select`, `Pan`만 편집 도구로 사용한다.
3. 객체 추가 도구는 후속 범위임을 확인한다.
4. Piping, Plate, 3D Solid, Artifact Lab, Case Study로 이동한다.

**완료 기준**

- 실제 동작하지 않는 제어를 활성 버튼으로 노출하지 않는다.
- 제품 경계와 `NOT A CERTIFICATION` 고지가 주 행동 근처에 남는다.
- workspace navigation의 현재 페이지가 시각·접근성 속성으로 표시된다.

## 이번 구현 범위

- compact quick-start rail과 상태 기반 주 행동
- 지원되는 `Select`·`Pan`과 후속 객체 생성 도구의 명확한 구분
- commit-on-blur/Enter, Escape 취소, inline validation을 갖는 수치 입력
- 결과 자동 이동·focus·live announcement
- violation 객체 선택과 `wall-north.end.x = 0mm` 데모 복구
- 복원 draft 안내와 undo 가능한 정상 샘플 복원
- 모바일 객체 선택기와 sticky action bar

## 비범위

- 자유 형상·새 wall/column/opening 생성
- 범용 자동수정 또는 LLM 기반 geometry 변경
- 구조·안전·법규 판정
- 계정, 서버 프로젝트 저장, 협업
- 다른 workspace의 정보 구조 전면 재작성

## 검증 지표

| 지표 | 목표 |
| --- | --- |
| 정상 샘플 첫 검증 | 샘플 선택 후 한 개의 주 행동으로 실행 |
| 실패에서 관련 객체 이동 | violation에서 한 번의 클릭 |
| 데모 실패 복구 | 한 번의 명시적 복구·재검증 행동 |
| 수치 입력 history | 한 번의 확정당 한 단계 |
| 결과 발견 | 검증 완료 후 자동 focus/scroll |
| 모바일 객체 선택 | 375px·820px에서 가능 |
| 미지원 제어 | 활성 dead control 0개 |
